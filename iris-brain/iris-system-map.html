◆ Memory — What Each System Remembers
Memory Layer IRIS Pro Local IRIS Core VPS
Conversation history ✅ Full context per session. Hooks auto-extract facts at session end. ⚠ In-memory Python dict. Lost on bot restart.
Session summaries ✅ Stored in memory/ + daily logs ✅ Mt. Everest summary saved to SQLite. Survives restarts.
Long-term memory ✅ 3-tier: Core State + daily logs + Pinecone (semantic search, optional) ❌ None. No Core State, no daily logs, no vector store.
Cross-session recall ✅ IRIS remembers past conversations, preferences, patterns ⚠ Can retrieve saved summary only. No conversational memory.
Telegram memory ✅ Pro bot handler feeds into mem0 + Pinecone ⚠ Last 20 messages in-memory. Lost on restart.
⚠ Known Gap: No Memory Bridge Between Core → Pro
When a free user upgrades from Core to Pro, Pro has zero knowledge of the Core conversation. The Mt. Everest summary sits in iris_core.db on the VPS — Pro never reads it. The user would need to re-share their goal with Pro, or we'd need to build a handoff mechanism (e.g., include the summary in the upgrade email so the user can paste it into their first Pro session).

This is a product decision, not a bug. The systems are intentionally separate for simplicity and security. Bridging them later is straightforward but not built yet.

◆ Key Architecture Insights
🔌
No Live Connection Between Pro & Core
Completely separate systems. Different servers, databases, AI engines, bot tokens. The only link is the upgrade URL. No API calls, no webhooks, no shared data between them.
🤖
Two Bots, One Personality
@IrisAccountabilityBot (Core, VPS) and @IrisProBot (Pro, local) are separate Telegram bots with separate tokens. Same IRIS voice — different scope. The bot you message determines what you get.
📄
Claude Code = 4 Interfaces, 1 Engine
CLI, Desktop App, Web App, and IDE extensions all run the same Claude Code engine against the same project directory. Same CLAUDE.md, same skills, same memory. Pick the interface that fits your workflow.
🐘
Memory is Pro-Only (for now)
Pro has rich 3-tier memory (core facts + daily logs + semantic vectors). Core has none beyond a saved summary. This is intentional — Core is lean for the free tier. But it means no memory carries over on upgrade.
📋
Core = Funnel, Pro = Product
Core gives away one free session (Mt. Everest) and converts to Pro. It's lean: one conversation, one summary, one email, one upgrade link. Pro is the full system with 38 skills, 7 MCP connectors, persistent memory, and a dashboard.
🔒
Everything is Self-Contained
No message queue. No shared DB server. No orchestrator. Each system is fully independent. Pro works without Core. Core works without Pro. Simplicity by design — complexity only where it earns its keep.
